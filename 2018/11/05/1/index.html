Mac运行环境配置

⛓Homebrew安装

目的: 使用Homebrew安装其他环境和软件.
流程: 打开terminal,执行下面1的命令.

  1. 输入 /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"命令.
  2. 看到提示后,按RETURN(回车键).
  3. 输入电脑密码.
  1. 自动下载安装命令行工具.
  1. 安装成功的提示.
  2. Homebrew的相关说明和命令.(官网: https://brew.sh/index_zh-cn.html)

⛓Node.js安装

目的: 安装Node.js环境.
流程: 去官网下载安装macOS的安装包(官网: https://nodejs.org/en/).

  1. 选择DOWNLOADS.
  2. 选择左侧稳定版本.
  3. 选择macOS Installer.
  4. 下载成功会在左下角显示.
  1. 双击下载的pkg包.



  1. 点击安装的时候, 会要求输入电脑密码.
  1. 如下图中1所示, 安装成功(使用node -v可以查版本号).
  2. 点击下图中2按钮, 会提示是否删除安装包, 直接删除就好.

⛓Git安装

目的: 安装Git版本控制.
流程: 打开terminal,执行brew install git命令.

  1. 输入 brew install git命令.
  2. 如下图中2所示,安装成功.

Atom安装和配置

⛓Atom安装(使用VSCode也可以)

目的: 安装Atom.
流程: 去官网下载安装.

  1. 官网: https://atom.io/.
  2. 点击下图中2所示,进行下载.
  3. 下载成功(双击解压).
  1. 点击下图中1所示,完成安装(自动打开IDE)

⛓Atom配置

目的: 安装Atom插件.
流程: 使用Atom Install a Package安装.

  1. 点击Install a Package.
  2. 点击Open Installer.
  1. 输入插件的名字.
  2. 点击Install.
  3. 下载并安装完成之后,会如下图中3所示.

需要安装的插件名称:

  1. atom-hexo hexo的插件,必须安装.
  2. platfromio-ide-terminal 命令行插件,可以再Atom中使用命令行,建议安装.
  3. tool-bar 和 tool-bar-markdown-writer, 可视化markdown工具,建议安装.
  4. markdown-preview-enhanced 实时显示blog界面效果,建议安装.
  5. atom-i18n 本地化的插件,英文看不懂的需要安装.
  6. 剩下的插件按照自己的爱好安装.

    4. post_asset_folder: true 设置图片的相对路径, 创建文章的时候, 会自动创建同名的资源文件夹.

  2. blog中themes中的指定主题(tranquilpeak)的_config.yml文件.

    7. auto_thumbnail_image: true 如果没有缩略图图像作为缩略图图像，则自动选择封面图像或帖子库中的第一张照片,会被子文件autoThumbnailImage属性覆盖.

  3. 详细博客的设置

    1. title: Mac + Atom + Hexo + hexo-theme-tranquilpeak主题 + GitHub + 全球IP 标题.
    2. #subtitle: 我是子标题, 子标题, 没发现在哪里显示.
    3. #帖子的时间 date: 2018-11-05 14:59:19.
    4. #tags: - Blog 标记.
    5. #thumbnailImage: 文章列表中单个文章显示的图片地址, 如果不需要定制,可以使用主题的通用设置,不需要使用此属性.
    6. #coverImage: 文章详情的banner图片地址, 如果不需要定制,可以使用主题的通用设置,不需要使用此属性.
    7. #coverCaption: 文章详情的banner图片的说明文字, 如果不需要定制,可以使用主题的通用设置,不需要使用此属性.
    8. #coverImage占据屏幕的高度, 默认是full, 如果不需要定制,可以使用主题的通用设置,不需要使用此属性.
    9. #coverSize: partial, 帖子的评论
    10. #帖子的日期,类别等 comments: true
    11. #meta: true, 导航,共享连接, #actions: true ,#mermaid: true

    13. #thumbnailImagePosition: left, 文章列表中单个文章显示的图片位置.#会覆盖全局thumbnail_image_position的值,默认是left,如果不需要更改,不需要重新给thumbnailImagePosition赋值.
    14. #autoThumbnailImage: true, 如果没有缩略图图像作为缩略图图像，则自动选择封面图像或帖子库中的第一张照片,会覆盖全局auto_thumbnail_image的值,默认是true,如果不需要更改,不需要重新给autoThumbnailImage赋值.
    15. #metaAlignment: center, 进入文章详情的时候, 标题 日期 类别显示的位置,默认是center,如果不需要更改,不需要重新给metaAlignment赋值.
    16. #coverMeta: in 在文章详情内,标题 时间是否显示在coverImage上面, in 在coverImage上面, out 在coverImage下面,默认是in,如果不需要更改,不需要重新给coverMeta赋值.

⛓Hexo命令

  1. hexo init folder 初始化命令, folder是想要创建的文件夹的名称, 建议使用blog.
  2. hexo new name 创建文章的命令, name是Project中的名字(类似于类名).
  3. hexo generate 简写为 hexo g, 生成静态文件的命令.
  4. hexo deploy, 部署网站的命令.
  5. hexo server, 开启本地服务的命令.
  6. hexo clean, 清除缓存(db.json)和已经生成的静态文件(public)的命令, 如果bolg不刷新, 可以执行此命令, 更换主题后, 必须执行此命令, 否则会造成blog不刷新的问题.

⛓主题安装

  1. hexo-theme-tranquilpeak 使用这个主题.
  2. 下载主题资源.
  3. 解压主题资源.

GitHub配置

必须有GitHub账号.

⛓Github仓库创建

目的: 创建Blog的仓库.
流程: 进入github个人界面.

  1. 点击Repositories
  2. 点击New

⛓Github仓库配置

目的: 进行仓库配置.
流程: 进入仓库配置界面

  1. Owner: 蓝色区域显示当前角色的用户名,可以选择使用指定角色来创建仓库(如果包含组织,也可以使用组织名来创建).
  2. Repository Name: 仓库名称, 必须使用 “用户名.github.io”, 用户名就是蓝色区域显示的名称.
  3. Description: 仓库的描述.
  4. 选择仓库是公开库,还是私有库,私有库是收费的.
  5. 初始化README描述文件.
  6. 选择库的语言类型,Blog会有很多种语言类型,所以直接使用None.
  7. 选择执照, MIT 或 Apache 或 None都可以.
  8. 点击Creat repository, 创建仓库.

⛓Github仓库 –> GitHub Pages设置

目的: 配置GitHub Pages.
流程: 进入Settings界面,配置GitHub Pages.

  1. 点击Settings
  1. 生成的博客地址,Blog发布后,可以直接访问.
  2. Source: 自己选择分支,可使用主分支.
  3. Theme Chooser: 不用选择.
  4. Custom domain: 全球IP,如果购买了全球IP,则在这个地方进行设置,可以直接通过全球IP访问Blog.
  5. 开启HTTPS.

⛓Blog生成地址替换

目的: Blog生成地址替换.
流程: 进入github个人界面,替换_config.yml中的repository的值.

  1. 点击Clone or download.
  2. 复制地址(例:https://github.com/13104495959/13104495959.github.io.git),去掉.git的后缀变成(例:https://github.com/13104495959/13104495959.github.io)
  1. 使用Atom打开blog文件夹,打开_config.yml文件.
  1. 替换黄色方框中的属性.
    1. type: 版本管理方式, 此博客中使用git.
    2. repository: 上面复制并且去掉后缀的地址(https://github.com/13104495959/13104495959.github.io).
    3. branch: 选择分支,此博客中使用主分支.

⛓上传Github仓库

  1. 进入blog文件, 执行 cd blog, 如果使用Atom打开blog文件夹, terminal中则不用执行次命令,默认在这个目录下.
  2. 打包文件, 使用命令hexo g.
  3. 上传文件, 使用命令hexo d.
  4. 输入Github账号(第一次需要).
  5. 输入Github密码(第一次需要).(如果需要删除账号密码,去钥匙串中删除即可)
  6. 上传成功,blog修改成功.
